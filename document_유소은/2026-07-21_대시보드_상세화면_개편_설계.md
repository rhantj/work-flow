# 대시보드 상세 화면 개편 (진행률/블로커/진행중/전체진행률)

- 날짜: 2026-07-21
- 브랜치: feature/fs3_ml-delay-risk
- 참고 시안: `dashboard_edited_v1.png` (팀 프로토타입 필기본)

## 목적

대시보드 상세 4개 화면(진행률 분석, 블로커 관리, 진행 중 업무 모니터링, 전체 진행률)의
카드/버튼/태그를 시안에 맞춰 교체한다. 지연 위험도는 이미 학습·배포된 ML 모델
(`delay_model.pkl`, HuggingFace Hub에서 `fetch_model.py`로 내려받아 `joblib.load()`로
로드하는 LightGBM 분류기)의 예측 결과(`정상`/`주의`/`위험`)를 그대로 소비한다 — 이번
작업 범위에서 모델/추론 파이프라인 자체는 변경하지 않는다.

## 범위

### 백엔드 (Spring, `backend_spring`)

1. `dashboard.DTO.DashboardTaskDto`에 `createdAt`, `updatedAt` (ISO-8601 문자열, null 가능)
   필드를 추가한다. `Task` 엔티티에는 이미 두 값이 존재하므로
   `DashboardService.toDashboardTaskDto()`에서 `DateTimeFormatter.ISO_LOCAL_DATE_TIME`로
   포맷해 채운다 (다른 매퍼들과 동일한 패턴).
2. `dashboard.DTO.ProgressDetailResponse`에 `projectDeadline`(LocalDate 문자열, null 가능),
   `projectCreatedAt`(날짜 문자열, null 가능) 필드를 추가한다. `DashboardService`에
   `ProjectRepository`를 주입해 `getProgressDetail()`에서 프로젝트를 조회해 채운다.
3. 그 외 지연 위험도 예측 관련 백엔드/ML 코드는 변경하지 않는다 (이미 동작 중).

### 프론트엔드 공용 (`App/frontend/src`)

1. `dashboard/libs/types/dashboard.ts` — 위 두 DTO에 대응하는 필드 추가.
2. `dashboard/libs/utils/dashboardTaskUtils.ts`에 헬퍼 추가:
   - `daysSince(dateStr: string | null): number | null` — ISO 날짜/일시 문자열과 현재 시각의
     차이를 "일" 단위로 반환 (자정 기준 X, 시간 포함 그대로 차이 계산 후 내림). `updatedAt`
     기반 체류일 계산에 사용.
   - `formatRelativeDate(dateStr: string | null): string` — `오늘`/`어제`/`n일 전`/(그 외엔
     `formatDashboardDueDate`와 동일한 `MM.DD`) 형식으로 변환.
   - 마감 D-Day 계산은 기존 `daysUntilDue()`를 프로젝트 마감일에도 그대로 재사용한다
     (별도 함수 추가 없음).
3. AI 챗봇 재사용을 위한 공용 이벤트 헬퍼를 신설한다:
   `ai/libs/utils/openAIAssistant.ts`
   ```ts
   export const OPEN_AI_ASSISTANT_EVENT = "workflow-ai:open-ai-assistant";
   export function openAIAssistant(question?: string): void {
     window.dispatchEvent(
       new CustomEvent(OPEN_AI_ASSISTANT_EVENT, { detail: { question, requestId: Date.now() } })
     );
   }
   ```
   - `AppShell.tsx`는 이 이벤트를 구독해 `aiOpen`을 켜는 동시에
     `pendingQuestion: { text: string; requestId: number } | null` 상태를 갱신하고,
     `<AIAssistant pendingQuestion={pendingQuestion} .../>`로 내려준다.
   - `AIAssistant.tsx`는 `pendingQuestion` prop을 받아 `requestId`가 바뀔 때마다
     기존 `send(text)`를 호출해 질문을 자동 전송한다 (동일 문구를 반복 요청해도
     `requestId`가 매번 달라지므로 정상 동작).
   - `DashboardView.tsx`의 기존 "AI 어시스턴트" 퀵액션과 `AppShell.tsx`에 중복 정의돼
     있던 이벤트명 문자열 리터럴은 이 공용 상수/헬퍼로 교체한다 (기능 변경 없음, 중복 제거).
   - 새 백엔드 엔드포인트는 만들지 않는다 — 기존 RAG 질의(`useRagQuery` → FastAPI)를 그대로 탄다.

## 페이지별 변경

### ProgressPage.tsx (진행률 분석)

- 상단 버튼 `AI 지연 분석` → `진행률 보고서 생성` (아이콘 `Sparkles` → `FileText`, 동일한
  보라색 그라디언트 스타일 유지).
- 클릭 시 `navigate(...)` 대신
  `openAIAssistant(`현재 프로젝트 진행률을 요약한 보고서를 작성해줘. 전체 ${totalTasks}건 중 ${doneTasks}건 완료(${progressPercent}%), 마일스톤 ${milestones.length}건.`)`
  호출.

### BlockersPage.tsx (블로커 관리)

1. 상단 카드 `7일 내 마감` → `평균 지연`.
   - 계산 대상: `progress.delayRisks` 중(이미 정상 제외, 프로젝트 전체 업무 기준)
     `dueDate`가 있고 이미 지난(`daysUntilDue(dueDate) < 0`) 항목만 필터.
   - 각 항목의 지연일 = `-daysUntilDue(dueDate)` (오늘 - 마감일, 일 단위).
   - 평균을 반올림해 `N일` 로 표시. 대상이 0건이면 `0일` / sub는 "지연 대상 없음".
   - sub(대상 있음): "주의·위험 {n}건 기준".
2. 블로커 카드 내부 `지연 위험` 태그(`isRisk` 뱃지) 색상을
   `bg-red-100 text-red-700 border-red-200` → `bg-orange-100 text-orange-700 border-orange-200`
   로 변경 (문구는 그대로).
3. 카드 우상단 D-Day 뱃지(`마감 미정`/`D-n`/`D+n`)를 전부
   `{daysSince(task.updatedAt)}일째 지속` 으로 교체 (마감일 유무와 무관하게 항상 체류일을
   표시 — 시안 요구사항이 이 뱃지 자체의 교체이므로).
4. 하단 `AI 분석 보기` 버튼 → `AI 해결 방법 추천`.
   - `onClick`: `navigate(...)` 대신
     `openAIAssistant(`'${task.title}' 업무가 블로커 상태입니다. 설명: ${task.description || "설명 없음"}. 이 블로커를 해결할 방법을 추천해줘.`)`.

### InProgressPage.tsx (진행 중 업무 모니터링)

모든 카드는 이 페이지에서 이미 필터링된 `inProgressTasks`(상태=진행중) 범위로 계산해
다른 카드들과 스코프를 맞춘다.

1. `높은 우선순위` 카드 → `업데이트 필요`: `daysSince(task.updatedAt) >= 3`인
   `inProgressTasks` 개수. sub: "3일 이상 정체".
2. `7일 내 마감` 카드 → `지연 위험`: `inProgressTasks` 중 `progress.delayRisks`에서
   `result`가 `위험`인 업무 개수 (기존 `riskTaskIds`는 Set이라 result를 모르므로,
   `taskId -> result` Map으로 교체).
3. `AI 위험 감지` 카드 → `D-Day`: `progress.projectDeadline` 기준
   `daysUntilDue(projectDeadline)` → `D-n`/`D+n`, sub에 마감일자(`formatDashboardDueDate`).
   `projectDeadline`이 없으면 `-` / sub "마감일 미설정".
4. 각 업무 카드 우측 정보 영역을 확장한다. 기존에는 `마감 {dueDate}` 뱃지 하나였던 것을
   세 줄짜리 작은 정보 블록으로 교체: `시작 {formatDashboardDueDate(createdAt)}` /
   `업데이트 {formatRelativeDate(updatedAt)}` / `마감 {formatDashboardDueDate(dueDate)}`
   (마감은 기존처럼 임박 시 강조색 유지).

### DashProgressPage.tsx (전체 진행률)

1. `미완료 업무` 카드 → `계획 대비`.
   - `projectCreatedAt`, `projectDeadline`이 모두 있으면
     `expected% = clamp((오늘 - projectCreatedAt) / (projectDeadline - projectCreatedAt) * 100, 0, 100)`.
   - `diff = round(expected - progressPercent)`.
   - value: `diff > 0 ? `-${diff}%p`` (계획보다 뒤처짐, 빨강) `: diff < 0 ? `+${-diff}%p`` (계획보다 앞섬, 초록) `: `0%p``.
   - sub: `예상 {expected}% · 실제 {progressPercent}%`.
   - 날짜 정보가 부족하면 value `-` / sub "마감일 미설정".
2. `지연 위험 업무` 카드 → `지연 업무` (라벨 문구만 교체, 계산 로직은 기존
   `delayRisks.length` 그대로 — 이미 정상 제외/주의+위험 합산).
3. `마일스톤` 카드 → `마감 D-day` (InProgressPage와 동일한 계산/포맷).
4. 상단 통계 카드 그리드에 `상태 범례` 카드를 신규 추가한다 (`grid-cols-4` →
   `grid-cols-5`). `DetailStatCard`는 단일 값 전용이라 재사용하지 않고, 동일한
   `bg-card rounded-xl border p-4` 톤의 작은 카드에 정상(초록)/주의(주황)/위험(빨강)
   색 점 + 라벨 3행을 나열한다.
