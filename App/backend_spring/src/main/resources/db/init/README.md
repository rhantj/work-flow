# 이 디렉터리는 "완전히 새 DB를 처음 만들 때"만 쓰입니다

`docker-compose.yml`이 이 폴더를 Postgres의 `docker-entrypoint-initdb.d`로 마운트해서,
**데이터가 하나도 없는 빈 볼륨으로 컨테이너를 처음 띄울 때만** 01부터 순서대로 자동
실행됩니다. 이미 데이터가 있는 볼륨을 재사용해 컨테이너만 재시작하면 이 폴더는 전혀
실행되지 않습니다.

**이미 존재하는(운영 중인) DB에 새 컬럼/테이블을 추가하는 스키마 변경이라면 여기가 아니라**
[`db/migration/`](../migration/)에 Flyway 규칙(`V<날짜>_<순번>__설명.sql`)으로 추가하세요.
Flyway는 앱이 기동할 때마다 실행되며 아직 적용 안 된 마이그레이션만 정확히 한 번씩
적용하므로, 이미 떠 있는 로컬/OCI DB에도 안전하게 반영됩니다. 자세한 내용은
[`App/DEPLOY_OCI.md`](../../../../../../DEPLOY_OCI.md) 8절 참고.

이 폴더에 파일을 추가해야 하는 경우는 사실상 "베이스 스키마 자체를 바꿀 때"(예:
`01_base_schema.sql` 갱신)뿐입니다 — 그 외에는 거의 항상 `db/migration/`이 맞는 위치입니다.

09(`09_drop_legacy_field.sql`)만은 예외입니다 — 완전히 빈 볼륨에서는 데이터도 구버전
인스턴스도 없어 안전하게 자동 실행되지만, 같은 내용을 이미 운영 중인 DB에 적용할 때는
`docs/db/migrations/011_drop_legacy_field.sql`을 통해 사람이 체크리스트를 확인한 뒤 수동으로
한 번만 실행합니다(`App/DEPLOY_OCI.md` 8-1절).
