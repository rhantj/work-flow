WorkFlow AI 데이터베이스(MySQL 기준)의 핵심 9개 테이블에 대한 논리 ERD와 물리 ERD를 직각 연결선과 1(││)·0..1(○│) 카디널리티 표기로 보여준다.

논리 ERD (Logical) — 핵심 엔터티 관계

타입 생략, 주요 속성만 표시 · 직각 연결선 + 1(││)·0..1(○│) 카디널리티

물리 ERD (Physical) — 컬럼·타입·키·카디널리티

MySQL 8.x 기준 테이블/컬럼/타입 · PK(주황)·FK(파랑) · 직각 연결선 + 1(││)·0..1(○│) 카디널리티

표기: ││ = 1(필수) · ○│ = 0..1(선택, FK NULL 허용). 모든 관계선 양끝에 이 막대 표기를 사용합니다. meetings↔meeting_analysis만 1:1 식별관계(meeting_id가 PK이자 FK)이고 나머지는 모두 1:N(자식 쪽은 여러 행을 가질 수 있음)입니다. tasks.milestone_id, tasks.assignee_id는 NULL 허용이라 해당 쪽만 0..1로 표시했습니다.
