# WorkFlow AI 개발 로드맵

## 1. 개발 전략

개발 기간은 약 1개월이며, 중간보고/2차보고/최종보고를 고려해 2주차 말까지 거의 완성본처럼 보이는 통합 데모를 확보한다.

전원 풀스택으로 개발하되, 각자 하나의 기능을 화면, API, DB, AI 연동까지 세로로 책임지는 기능 오너제로 운영한다.

## 2. 7인 풀스택 역할 분담

| 오너 | 담당 기능 | 개발 범위 |
| --- | --- | --- |
| FS-1 | 인증/프로젝트/RBAC | 로그인, 프로젝트 생성, 팀원 초대, 역할 권한 |
| FS-2 | 회의록 AI/To-Do | 업로드, AI 요약, To-Do 생성, 역할 배정 |
| FS-3 | 대시보드/ML 위험도 | 진행률, 마감 임박, 지연 위험도 예측, 업무 편중 점수 |
| FS-4 | AI Assistant/RAG | 프로젝트 Q&A, embedding, vector search, 출처 표시 |
| FS-5 | 산출물 생성 | 발표자료, 보고서, README, 제안서, 실험 보고서 |
| FS-6 | 업무 보드/GitHub | 카테고리별 업무 UI, GitHub 커밋/PR 연동 |
| FS-7 | 심사자/기여도/QA | 기여도 리포트, AI 평가 근거, 평가 점수, 테스트 |

## 5. ML/DL/LLM 구현 로드맵

| 단계 | 모델 | 구현 방식 | 발표 포인트 |
| --- | --- | --- | --- |
| 1단계 | 지연 위험도 예측 | scikit-learn/LightGBM baseline | 업무 데이터로 위험도를 예측한다 |
| 1단계 | 업무 편중 점수 | 회귀/분류 모델 또는 scoring + ML 비교 | 팀원 과부하를 정량화한다 |
| 2단계 | 회의록 문장 분류 | KoBERT/KLUE-BERT fine-tuning 또는 zero-shot 비교 | 회의 내용을 결정사항/할 일/위험으로 분류한다 |
| 2단계 | STT | Whisper API 또는 local Whisper | 음성 회의도 업무로 변환한다 |
| 3단계 | 기여도 이상치 탐지 | Isolation Forest | 무임승차/과도한 편중 패턴을 탐지한다 |
| 3단계 | RAG Assistant | Embedding + Vector DB + LLM | 프로젝트 근거 기반 답변을 제공한다 |

## 6. 발표 3회 전략

| 발표 | 시점 | 핵심 메시지 |

| 2차보고 | 2주차 말 | 회의록에서 업무, 대시보드, AI 예측, 산출물까지 이어지는 통합 흐름 |
| 최종보고 | 4주차 말 | ML/DL/LLM을 실제 기능에 녹인 AI 협업 플랫폼 |

## 7. 최종 데모 시나리오

1. 팀장으로 로그인한다.
2. 회의록 AI에서 문서 회의록을 업로드한다.
3. AI가 요약, 결정사항, 위험요소, To-Do를 생성한다.
4. 미배정 업무를 팀장이 담당자에게 배정한다.
5. 업무 보드에 업무가 생성된다.
6. 대시보드에서 진행률과 ML 지연 위험도를 확인한다.
7. AI Assistant에게 "이번 주 위험한 업무가 뭐야?"라고 질문한다.
8. 산출물 생성에서 발표자료 초안을 생성한다.
9. 심사자로 전환해 개인별 기여도 리포트와 AI 평가 근거를 확인한다.
10. 최종 평가 점수 공개 설정을 보여준다.
