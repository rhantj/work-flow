# 심사자 가입 신청 UI — 백엔드 부분 연동 상태

## 상태
부분 해결 (Google OAuth는 실제 연결됨, 심사자 전용 데이터는 여전히 미저장)

## 요약
`App/frontend/src/auth/screen/SignupScreen.tsx`에 "교수/심사자로 가입 신청" 체크박스와 소속/교수 인증번호/인증서류 첨부 폼을 추가했다 (`feature/ai_assistent` 브랜치, 커밋 `2cf6ced`). 최초 구현 시에는 `handleSignup`이 서버 요청 없이 가짜 성공 메시지만 표시했으나, 이후 **심사자 체크 여부와 관계없이 실제 `loginWithGoogle()` (백엔드 `/api/v1/auth/google` → Google OAuth → `/api/v1/auth/google/callback`)을 호출하도록 수정**했다.

## 현재 동작

- **Google OAuth 연결: 완료.** 체크박스 선택 여부와 무관하게 버튼 클릭 시 실제 Google 로그인/가입 플로우로 리다이렉트된다. (기존 `/login`, `/signup` 일반 가입과 동일한 백엔드 엔드포인트 재사용)
- **심사자 전용 입력값(소속/교수 인증번호/인증서류): 여전히 미저장.** 클라이언트 상태로만 존재하다가 `loginWithGoogle()` 호출과 함께 페이지가 이동하면서 소실된다. 서버로 전송되지 않는다.
- **검증: 클라이언트 단순 공백 체크만 존재.** 소속/인증번호가 비어 있으면 제출을 막지만, 형식 검증이나 서버 검증은 없다.
- **파일 업로드: 미구현.** `handleCertificateChange`는 `File.name`만 상태에 저장하며, 실제 업로드는 발생하지 않는다.
- **안내 문구:** "입력한 인증 정보는 Google 가입 후 관리자가 별도로 확인합니다. (자동 승인 아님)" — 오해를 유발하던 기존의 가짜 "접수 완료" 성공 메시지는 제거했다.

## 남은 갭

- `docs/projects/WorkFlow_AI_인증_RBAC_구현_파일.md` 4장에 기록된 대로, 가입 시점에 "일반 유저 / 심사자(교수 학번 인증)" 분기 자체가 백엔드에 없다. 현재 `REVIEWER` 역할은 프로젝트 리더의 초대(`OnboardingScreen.tsx`)를 통해서만 부여된다.
- 심사자 신청 시 입력한 소속/교수 인증번호/서류는 Google 가입 완료 후에도 어디에도 남지 않는다 — 사실상 이 입력 폼은 아직 장식적 UI에 가깝다.

## 필요한 후속 작업 (미착수)

- [ ] 심사자 가입 신청용 API 엔드포인트 설계/구현 (예: `POST /api/v1/auth/reviewer-applications`) — Google 가입 완료 후 인증된 사용자 기준으로 소속/인증번호를 저장
- [ ] 인증서류 파일 업로드 처리 (스토리지 연동)
- [ ] 서버 측 입력 검증 (소속, 교수 인증번호 형식/유효성)
- [ ] 관리자 승인 워크플로우 (승인/반려 상태 관리, 승인 시 `REVIEWER` 역할 부여)
- [ ] 프론트엔드에서 Google 가입 완료 후 위 엔드포인트로 심사자 신청 데이터를 별도 전송하도록 연결
